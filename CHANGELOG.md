# 库存管家 — 更新日志

## v1.1 (2026-07-01) — 后端+数据库+权限+图表+批量操作

### 新增
- **Express + SQLite 后端** (`server/server.ts`，400+ 行)
- **REST API**：用户认证、商品 CRUD、出入库、分类、日志、看板统计、趋势图表、用户管理、库存盘点、图片上传
- **JWT 认证**：token 7 天有效期，登录后多端数据实时互通
- **数据库**：SQLite 单文件，users / categories / products / stock_logs 四表，自动初始化演示数据
- **用户管理页**：管理员可查看所有用户、修改角色、重置密码、删除用户
- **权限控制**：操作员只能入库出库，不能删除商品、编辑分类、管理用户
- **看板趋势图表**：近 7 天 / 30 天出入库柱状图 + 分类商品数占比饼图
- **批量删除商品**：表格多选 → 一键批量删除
- **库存盘点 API**：批量提交实际数量，自动计算差异并修正
- **商品图片上传 API**：multer 文件存储
- **看板卡片点击跳转**：商品总数 → 商品管理 / 预警商品 → 库存预警
- **服务端托管前端**：Express 同时提供 API 和静态页面，一个端口 (3001) 搞定
- **手机扫码二维码**：`qrcode.png`
- **.node-version**：指定 Node 22，适配 Render.com 云部署

### 修改
- 全部页面从 localStorage 迁移到后端 API，字段名适配 (`category_id` / `min_stock` / `created_at`)
- 侧边栏导航按角色过滤：操作员不显示分类管理、用户管理
- 商品列表 / 网格视图切换修复：桌面端切换生效
- 登录 / 注册改为异步调用后端 API
- `vite.config.ts` 添加开发代理 (`/api` → `localhost:3001`)

### 架构
- 前端：`http://localhost:3001`（Express 静态服务）或 `http://localhost:5199`（Vite 开发模式）
- 后端：Express + better-sqlite3 单文件数据库
- 部署：Render.com Node.js Web Service
